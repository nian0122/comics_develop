# 漫画阅读器项目需求文档

**版本**: 1.0  
**更新日期**: 2026-05-10  
**状态**: 已实现

---

## 1. 项目概述

### 1.1 项目目标

构建一个本地漫画阅读器，支持图片和视频混合阅读，提供流畅的阅读体验和高效的管理工具。

**核心目标**:

- 支持本地漫画文件的高性能浏览和阅读
- 支持 HQ/LQ 双模式图片加载，平衡画质和加载速度
- 支持图片和视频（MP4/MOV/GIF）混合阅读
- 提供图片优化、文件管理等工具
- 移动端优先的响应式设计

### 1.2 目标用户

- 本地漫画收藏用户
- 需要离线阅读的用户
- 需要管理大量漫画文件的用户

### 1.3 使用场景

- **场景1**: 用户在电脑或手机上浏览本地漫画系列列表
- **场景2**: 用户逐级进入目录结构，找到想阅读的章节
- **场景3**: 用户纵向滚动阅读章节内容
- **场景4**: 用户需要批量将 HQ 原图压缩为 LQ WebP 格式
- **场景5**: 用户需要清理或管理漫画文件

## 2. 功能需求

### 2.1 漫画浏览功能

#### FR-001: 系列列表展示

**优先级**: P0  
**描述**: 展示所有漫画系列列表

**验收标准**:

- 后端扫描 HQ 根目录下的一级目录作为系列
- 使用自然排序（1, 2, 10 而非 1, 10, 2）
- 支持搜索过滤
- 支持中文系列名称
- 系列列表缓存到 Redis（`v2:series_list`）

#### FR-002: 逐级目录浏览

**优先级**: P0  
**描述**: 用户逐级浏览系列内的目录结构

**验收标准**:

- 只展示当前层级的直接子目录
- 混合显示目录节点和章节节点
- 目录节点可继续展开
- 章节节点显示首图卡片
- 按自然排序排列
- 层级数据缓存到 Redis（`v2:level:{series}[:{path}]`）

#### FR-003: 章节首图预览

**优先级**: P0  
**描述**: 章节列表显示首图缩略图

**验收标准**:

- 章节首图使用自然排序后的第一张普通图片
- 支持 LQ/HQ 双模式加载
- LQ 存在时优先使用 LQ
- 纯视频/GIF 章节显示"无预览"

#### FR-004: 章节文件列表

**优先级**: P0  
**描述**: 获取章节内所有媒体文件

**验收标准**:

- 返回支持格式的文件列表（jpg, jpeg, png, webp, mp4, mov, gif）
- 包含文件元数据：文件名、基础名、媒体类型、HQ/LQ 状态
- 返回 HQ/LQ URL
- 视频文件返回视频 URL
- 按自然排序排列
- 数据缓存到 Redis（`v2:chapter_files:{series}:{path}`）

### 2.2 阅读功能

#### FR-005: 纵向阅读流

**优先级**: P0  
**描述**: 图片和视频纵向排列，用户上下滑动阅读

**验收标准**:

- 图片和视频按章节文件顺序纵向排列
- 图片宽度适配屏幕
- 图片之间最小间距
- 支持懒加载

#### FR-006: 智能图片源

**优先级**: P0  
**描述**: 根据章节首图的 LQ/HQ 存在情况，决定阅读器默认加载源

**验收标准**:

- 如果章节首图有 LQ，则该章节默认使用 LQ 加载
- 如果章节首图只有 HQ，则该章节默认使用 HQ 加载
- 单张图片加载失败时回退到另一源
- 双击图片可切换 HQ（如果 HQ 可用）

#### FR-007: 阅读进度显示

**优先级**: P1  
**描述**: 显示当前阅读进度

**验收标准**:

- 格式：当前页 / 总页数
- 根据视口中心位置计算当前页
- 实时更新
- 位置：底部居中半透明

#### FR-008: 章节导航

**优先级**: P1  
**描述**: 阅读页支持上一话/下一话导航

**验收标准**:

- 阅读页悬浮按钮展开导航菜单
- 上一话/下一话按钮
- 返回目录按钮
- 边界章节禁用对应导航按钮
- 返回目录后保持当前章节高亮

#### FR-009: 本地阅读进度保存

**优先级**: P1  
**描述**: 保存用户阅读进度到本地存储

**验收标准**:

- 章节级进度保存（阅读到第几页）
- 系列级进度保存（阅读到哪个章节）
- 本地存储持久化
- 支持跨设备同步（可选）

### 2.3 工具管理功能

#### FR-010: 图片优化器

**优先级**: P1  
**描述**: 批量将 HQ 图片转换为 LQ WebP

**验收标准**:

- 支持按系列或自定义目录扫描
- 支持设置并发数和 WebP 质量
- 支持强制重处理
- 异步执行，支持状态查询
- 执行日志实时返回

#### FR-011: 清空文件内容

**优先级**: P2  
**描述**: 将文件截断为 0 字节，保留文件名但清空内容

**验收标准**:

- 支持按扩展名过滤
- 支持设置最小文件大小
- 支持干跑模式预览
- 异步执行，支持状态查询

#### FR-012: 叶目录图片查找

**优先级**: P2  
**描述**: 查找目标路径下所有叶目录的第一张图片

**验收标准**:

- 支持按扩展名过滤
- 支持 JSON 格式输出
- 异步执行，支持状态查询

#### FR-013: 工具执行管理

**优先级**: P1  
**描述**: 管理工具执行状态

**验收标准**:

- 查看所有执行记录
- 查询单个执行状态
- 取消正在执行的工具
- 清理已完成记录
- 实时查看执行日志

## 3. 非功能需求

### 3.1 性能需求

#### NFR-001: 响应时间

**优先级**: P0

- 系列列表加载：< 500ms（缓存命中）
- 章节文件列表加载：< 1s（缓存命中）
- 图片首帧显示：< 200ms（LQ）
- 目录层级切换：< 300ms（缓存命中）

#### NFR-002: 并发能力

**优先级**: P1

- 支持多用户同时浏览（通过 Nginx 反向代理）
- 工具执行使用虚拟线程池，不阻塞主线程
- 图片加载支持浏览器并发

#### NFR-003: 资源占用

**优先级**: P1

- 内存缓存策略防止内存溢出
- 支持 LRU 缓存淘汰
- Redis 缓存 TTL：604800 秒（7天）

### 3.2 可用性需求

#### NFR-004: Redis 降级

**优先级**: P0  
**描述**: Redis 不可用时自动降级到文件系统扫描

**验收标准**:

- Redis 连接失败时不影响服务
- 自动降级到文件系统扫描
- 日志记录降级事件
- Redis 恢复后自动恢复缓存

#### NFR-005: 文件系统容错

**优先级**: P0

- 目录不存在时返回空数据而非错误
- 文件不存在时返回 404 或空响应
- 文件名编码问题正确处理
- 并发访问文件系统安全

### 3.3 安全需求

#### NFR-006: 路径安全

**优先级**: P0  
**描述**: 防止路径遍历攻击

**验收标准**:

- 所有用户输入路径经过 normalize
- 检查路径是否以系列根目录开头
- 禁止 `..` 逃逸出授权目录
- 非法路径返回 400 错误

#### NFR-007: CORS 配置

**优先级**: P1

- API 支持跨域请求
- 静态资源支持跨域
- 视频文件支持 Range 请求

### 3.4 兼容性需求

#### NFR-008: 浏览器兼容

**优先级**: P1

- Chrome/Edge/Firefox/Safari 最新版本
- 移动端浏览器支持
- 响应式布局适配

#### NFR-009: 文件格式兼容

**优先级**: P0

**支持格式**:

- 图片：JPG, JPEG, PNG, WebP
- 视频：MP4, MOV
- 动图：GIF（走 video 接口）

### 3.5 可维护性需求

#### NFR-010: 日志记录

**优先级**: P1

- 使用 SLF4J + Logback
- 日志级别：ERROR, WARN, INFO, DEBUG
- 关键操作记录：扫描、缓存、工具执行
- 日志文件轮转

#### NFR-011: 监控指标

**优先级**: P2

- 缓存命中率
- 文件扫描耗时
- 工具执行统计
- 错误率监控

## 4. 数据需求

### 4.1 漫画目录结构

```text
COMICS_ROOT/
├── h_photograph/{series}/{chapter}/     # HQ 原图
└── l_photograph/{series}/{chapter}/     # LQ WebP 压缩图
```

**命名约定**:

- 系列名：文件夹名
- 章节名：文件夹名（支持多级）
- 文件名：自然排序友好（001.jpg, 002.jpg...）

### 4.2 缓存数据结构

**系列列表** (`v2:series_list`):

```json
["系列A", "系列B"]
```

**章节文件** (`v2:chapter_files:{series}:{path}`):

```json
{
  "path": "第01话",
  "files": [...],
  "total": 36
}
```

**层级节点** (`v2:level:{series}[:{path}]`):

```json
{
  "path": "合集/卷1",
  "nodes": [...]
}
```

### 4.3 本地存储

**阅读进度**:

- 键：`reader_progress_{series}_{chapter}`
- 值：`{"page": 12, "total": 36, "timestamp": 1234567890}`

**系列进度**:

- 键：`series_progress_{series}`
- 值：`{"chapter": "第01话", "page": 12, "timestamp": 1234567890}`

## 5. 界面需求

### 5.1 页面结构

```text
系列列表页 → 目录浏览页 → 阅读页
     ↑           ↑
     └───────────┘
```

### 5.2 响应式设计

**断点**:

- 移动端：< 768px
- 平板：768px - 1024px
- 桌面：> 1024px

**移动端优先**:

- 首页：单列系列列表
- 目录页：单列目录 + 横向章节卡片
- 阅读页：全屏纵向阅读流

### 5.3 视觉设计

**章节卡片**:

- 横向布局：首图 + 信息
- 首图：左侧，固定宽度
- 信息：章节名、页数、阅读进度
- 当前阅读章节高亮
- 已读完显示完成标记

**阅读页**:

- 图片/视频纵向排列
- 最小间距
- 宽度自适应
- 底部进度指示器

## 6. 部署需求

### 6.1 部署模式

**本地开发**:

- 后端：Spring Boot，端口 500
- 前端：Vite Dev Server，端口 3000
- Redis：Docker，端口 6379

**Docker 部署**:

- Nginx：端口 5000（前端 + 静态资源代理）
- 后端：端口 500（Nginx 反向代理到 `/api`）
- Redis：内部端口 6379

### 6.2 配置需求

| 配置项 | 环境变量 | 默认值 | 说明 |
|--------|----------|--------|------|
| 漫画根目录 | `COMICS_ROOT_DIR` | `F:\games\comics` | HQ/LQ 父目录 |
| HQ 子目录 | `HQ_SUB_DIR` | `h_photograph` | 原图目录 |
| LQ 子目录 | `LQ_SUB_DIR` | `l_photograph` | WebP 目录 |
| 工具根目录 | `TOOL_ROOT_DIR` | `D:/projects/.../tools` | Go CLI 工具目录 |
| 缓存 TTL | `CACHE_EXPIRATION` | `604800` | Redis TTL 秒数 |
| Redis 主机 | `REDIS_HOST` | `localhost` | Redis 地址 |
| Redis 端口 | `REDIS_PORT` | `6379` | Redis 端口 |
